我看过仓库里的 `scrcpy` 和 `ws-scrcpy` 相关源码了，先给你一个能直接建立整体认知的版本。

`scrcpy` 本体是一个典型的“双端架构”：
1. 桌面端客户端：C 写的原生程序，负责启动 `adb`、把服务端推到手机上、建立连接、解码音视频、显示画面、转发键鼠事件。这个结构在 [develop.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/develop.md:5) 讲得很清楚。
2. Android 端服务端：Java 写的服务，运行在手机的 `shell` 权限下，负责采集屏幕、编码视频/音频、接收控制消息并注入输入事件，见 [develop.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/develop.md:63)。

它的核心原理可以概括成一句话：  
**PC 端通过 adb 把 `scrcpy-server.jar` 推到设备上运行，设备把编码后的画面/声音通过 socket 发回 PC，PC 再解码显示；控制则反方向通过 control socket 发回手机。**

更细一点：
- `video`、`audio`、`control` 是分离的 socket，1 到 3 条连接按功能拆开，[develop.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/develop.md:12)。
- 视频默认是 H.264，服务端用 `MediaCodec` 编码，客户端用 FFmpeg 解码，[develop.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/develop.md:159)。
- 音频也是设备端采集编码、PC 端解码播放，[develop.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/develop.md:187)。
- 控制消息通过 `Controller` 处理，最终调用 Android 隐藏接口 `InputManager.injectInputEvent()` 注入触摸/键盘/鼠标事件，[develop.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/develop.md:199)。
- 一个很关键的点是：应用层上手机是“服务端”，但默认网络层上反过来是 **PC 先监听端口，手机主动连回来**，这样能减少连接竞争和时序问题，[develop.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/develop.md:43)。

从源码组织上看也很清楚：
- 客户端源码入口和模块列表在 [app/meson.build](/D:/projects/Cloud-Phone/projects/scrcpy/app/meson.build:1)，里面能看到 `adb`、`decoder`、`demuxer`、`controller`、`recorder`、`scrcpy.c` 等模块。
- 顶层项目版本和编译分支在 [meson.build](/D:/projects/Cloud-Phone/projects/scrcpy/meson.build:1)。
- Android 服务端的 Gradle 配置在 [server/build.gradle](/D:/projects/Cloud-Phone/projects/scrcpy/server/build.gradle:1)，当前这份是 `versionName "4.0"`，`minSdkVersion 21`。

构建方式分两部分：

`scrcpy` 本体构建：
- 客户端用 **Meson + Ninja** 构建；
- Android 服务端用 **Gradle + Android SDK + JDK 17** 构建；
- 官方文档在 [build.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/build.md:20)。

常见流程是：
1. 安装 `adb`、FFmpeg、SDL3；
2. 配好 `ANDROID_SDK_ROOT`；
3. 执行 Meson 配置和 Ninja 编译。具体命令说明在 [build.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/build.md:208)。

还有一个很实用的模式：
- 只编客户端，不自己编 Android 服务端，直接用预编译 `scrcpy-server`；
- 这也是官方支持的，见 [build.md](/D:/projects/Cloud-Phone/projects/scrcpy/doc/build.md:237)。

你仓库里另外还有一个 `ws-scrcpy`，它不是官方原生桌面版，而是 **给 scrcpy 套了一层 Web 化方案**：
- 浏览器负责显示和交互；
- Node.js 作为中间层；
- Android 端使用改造过的 `scrcpy-server.jar`，支持 WebSocket；
- 整体关系在 [scheme.md](/D:/projects/Cloud-Phone/projects/ws-scrcpy/docs/scheme.md:1)。

它的构建方式和原生版不同：
- 前后端都在 Node/TypeScript 体系里；
- 用 `webpack` 打包，`npm start` 启动；
- 构建脚本定义在 [package.json](/D:/projects/Cloud-Phone/projects/ws-scrcpy/package.json:5)；
- 说明文档在 [ws-scrcpy README](/D:/projects/Cloud-Phone/projects/ws-scrcpy/README.md:25)。

`ws-scrcpy` 的原理是：
1. Node 端通过 `adb` 管理设备；
2. 浏览器和 Node 用 WebSocket 通信；
3. 改造过的 Android `scrcpy` 服务端输出 H264；
4. 浏览器用 MSE / WebCodecs / wasm 解码器播放，[README.md](/D:/projects/Cloud-Phone/projects/ws-scrcpy/README.md:46)。
